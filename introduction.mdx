# Welcome to Quest

Quest is a Retrieval-Augmented Generation (RAG) engine designed to assist with solving and explaining Data Structures and Algorithms (DSA) problems. It leverages a combination of retrieval-based methods and generative models to provide accurate and context-aware solutions to coding problems, explanations of concepts, and metadata about LeetCode-style questions.

## What is Quest?

Quest combines the power of semantic search with large language models to provide intelligent assistance for coding problems. Instead of relying solely on web searches or generic AI responses, Quest retrieves relevant solutions from a curated dataset of 1800+ LeetCode problems and generates context-aware explanations.

## What problems does Quest solve?

<AccordionGroup>
  <Accordion title="Exact problem matching">
    Quest uses a hash map for instant retrieval of exact problem matches, returning solutions in milliseconds rather than seconds.
  </Accordion>
  <Accordion title="Semantic search for similar problems">
    When exact matches aren't found, Quest uses FAISS and HNSW indexing to find semantically similar problems and solutions.
  </Accordion>
  <Accordion title="Context-aware explanations">
    Quest doesn't just return code—it generates detailed explanations incorporating conversation history and retrieved context.
  </Accordion>
  <Accordion title="On-device processing">
    All processing happens locally using Ollama, ensuring your code and queries remain private.
  </Accordion>
</AccordionGroup>

## How it works

Quest uses a multi-stage retrieval and generation pipeline:

1. **Query normalization** - Normalizes user queries to check for exact matches
2. **Exact matching** - Uses hash map lookup for instant exact problem matches
3. **Semantic retrieval** - Encodes query using `all-MiniLM-L6-v2` and searches HNSW index
4. **Confidence filtering** - Filters results based on similarity scores (default threshold: 0.6)
5. **Prompt generation** - Creates context-aware prompts incorporating conversation history
6. **LLM generation** - Generates responses using Ollama with appropriate model and parameters
7. **Response filtering** - Extracts answer from reasoning traces for deepseek-r1 model

<Note>
Quest processes everything on your local machine. You'll need to pull the required Ollama models before using the engine.
</Note>

## Architecture

The core components of Quest include:

- **LeetCodeRetriever** (`retriever2.py:26`) - Manages FAISS HNSW index and metadata for 1800+ solutions
- **RAGEngine** (`rag_engine3.py:18`) - Orchestrates retrieval, prompt generation, and LLM calls
- **ConversationHistory** (`memory_buffer.py:5`) - Maintains conversation context with configurable history limit
- **PromptTemplates** - Provides model-specific prompt templates for general and reasoning modes
- **Flask API** (`app.py:9`) - REST API with endpoints for search, mode switching, and history management

## Dataset

Quest includes a pre-generated dataset of **1800+ LeetCode solutions** in JSON format, created using the `qwen2.5-coder:1.5b` model. Each solution includes:

- Complete code implementation
- Problem difficulty level
- Related topics and concepts
- Companies that ask the problem
- Approach explanations

<Tip>
The solutions dataset is compressed in `leetcode_solutions.zip`. You'll need to unzip it during installation.
</Tip>
